## 🧠 【必读】核心认知篇：Vibe Coding思维

*(在开始Day 1之前，请先升级你的大脑)*

很多人学Vibe Coding，只停留在"让AI写代码"阶段。真正的创作者，必须掌握从想法到产品的**全链路控制权**。

**从想法到产品（Product Thinking）：**

以前你是在写代码，现在你要学的是产品思维。AI不懂"做一个好玩的游戏"，但它懂"实现用户登录功能，需要用户名、密码输入框和提交按钮"。

**与AI协作的艺术（AI Collaboration）：**

AI不是魔法，它只是一个非常强大的执行者。你告诉它做什么，它就做什么。如果你自己都不知道要做什么，AI自然也无法帮你做出理想的产品。

**迭代的节奏（Iteration Rhythm）：**

开发不是一次性完成的。你需要理解"小步快跑"，什么时候该添加新功能？什么时候该优化体验？什么时候该修复Bug？

## Day 3：理解项目开发流程

### 🎯 学习目标：掌握标准Vibe Coding开发流程

做项目不是打开AI就开始写代码，而是要遵循一套标准流程。方法和顺序很重要，**先想清楚再动手**，可以让AI更准确理解你的意图，减少返工和无效生成。

今天我们来学习这套经过验证的**5步工作流**。

### 📋 5步开发流程

```Markdown
┌─────────────────────────────────────┐
│                    Vibe Coding 5步工作流                      │
├─────────────┬─────────────┬─────────┤
│  需求研究   │   PRD文档   │  技术设计   │  AGENTS.md  │ 开发 │
│  Research  │    文档     │   文档     │   指令文件    │ 迭代 │
└─────────────┴─────────────┴─────────┘
```

**Step 1：需求研究（Research）**

在动手之前，先搞清楚：

- 我到底要做什么？
- 为什么要做？
- 有没有人做过类似的东西？

输出：`RESEARCH.md`

**Step 2：产品需求文档（PRD）**

明确要做哪些功能，不做哪些功能：

- 产品概述
- 目标用户
- 核心功能列表
- 功能优先级（MVP vs 后续版本）
- 界面设计要求

输出：`PRD.md`

**Step 3：技术设计文档（Tech Design）**

确定用什么技术来实现：

- 技术栈选择
- 项目结构
- 数据模型
- 关键技术点

输出：`TECH_DESIGN.md`

**Step 4：AI代理指令（AGENTS.md）**

告诉AI在这个项目中应该遵循什么规则：

- 开发规范
- 代码风格
- 测试要求
- 注意事项

输出：`AGENTS.md`

**Step 5：实现和迭代（Build）**

小步快跑，逐步迭代：

1. 生成基础框架
2. 逐步实现核心功能
3. 优化实现细节

# 第二阶段：需求与设计（Day 4-6）

## Day 4：撰写产品需求文档（PRD）

### 🎯 学习目标：学会写一份专业的PRD

PRD（Product Requirements Document）是整个项目**最重要**的文档。**一份好的PRD可以让AI准确理解你的意图**，生成高质量的代码。

### 📝 PRD模板

```Markdown
# AI海龟汤游戏 PRD

## 产品概述
一个AI驱动的海龟汤游戏网站，玩家可以与AI主持人对话，通过提问推理出故事真相。

## 目标用户
- 喜欢推理游戏的玩家
- 想要轻松娱乐的上班族
- 对AI应用感兴趣的技术爱好者

## 核心功能

### MVP（必须做）
1. **游戏大厅**
   - 显示游戏列表（不同难度/类型的海龟汤）
   - 点击开始游戏

2. **游戏页面**
   - 显示汤面故事
   - 聊天输入框，可以输入问题
   - AI只回答"是"、"否"、"无关"
   - 显示对话历史
   - "查看汤底"按钮
   - "结束游戏"按钮

3. **汤底页面**
   - 显示完整汤底
   - 显示玩家推理过程
   - "再来一局"按钮

### 后续可以做
- 用户登录系统
- 游戏记录保存
- 排行榜
- 自定义海龟汤
- 多人对战模式

## 界面设计要求
- 整体风格：神秘悬疑，深蓝色调
- 游戏大厅：卡片式布局，显示故事标题和难度
- 游戏页面：聊天界面风格，类似微信
- 汤底页面：揭晓真相的仪式感

## 技术栈建议
- 前端：React + TypeScript + Tailwind CSS
- 后端：Node.js + Express
- AI：DeepSeek / Claude API
- 部署：Vercel
```

## Day 5：技术设计与架构

### 🎯 学习目标：学会做技术选型与架构设计

有了PRD，我们要确定用什么技术来实现这些功能。

**技术上的新同学不必有心理压力，按步骤做的同时可以把不懂的问题（比如前后端工具、框架、文档格式；比如路由；比如API）发给AI（GPT/Deepseek)让它详细解释。**

### 🏗️ 技术设计文档模板

```Markdown
# AI海龟汤游戏技术设计

## 技术栈
- 前端：React + TypeScript + Vite
- 样式：Tailwind CSS
- 状态管理：React Hooks（useState, useContext）
- 路由：React Router
- 后端：Node.js + Express（可选，前期可用前端模拟）
- AI API：DeepSeek / 智谱AI（比较推荐Deepseek，token很便宜）
## 项目结构
src/components/ # 组件
GameCard.tsx    # 游戏卡片
ChatBox.tsx     # 聊天框
Message.tsx     # 单条消息
StoryReveal.tsx # 汤底揭晓
pages/          # 页面
Home.tsx        # 首页/游戏大厅
Game.tsx        # 游戏页面
Result.tsx      # 结果页面
data/           # 数据
stories.ts      # 海龟汤故事数据
App.tsx
main.tsx
## 数据模型

### Story（海龟汤故事）
- id: string
- title: string
- difficulty: 'easy' | 'medium' | 'hard'
- surface: string（汤面）
- bottom: string（汤底）

### Message（对话消息）
- id: string
- role: 'user' | 'assistant'
- content: string
- timestamp: number

## 核心流程
1. 玩家选择故事 → 显示汤面
2. 玩家输入问题 → 发送给AI
3. AI根据汤底判断 → 返回"是/否/无关"
4. 重复2-3步 → 直到玩家猜出真相或放弃
5. 显示汤底 → 可以选择再来一局
## AI Prompt设计

你是一个海龟汤游戏的主持人。

当前故事的汤面是：{surface}
故事的汤底是：{bottom}

玩家会向你提问，你只能回答以下三种之一：
1."是"：玩家的猜测与汤底一致
2."否"：玩家的猜测与汤底矛盾
3."无关"：玩家的猜测与汤底无关，无法判断

注意：
1.严格根据汤底判断，不要额外推理
2.只回答"是"、"否"、"无关"，不要解释
3.保持神秘感，不要透露汤底

玩家问：{question}
请回答：
```

## Day 6：创建AI开发指令（AGENTS.md）

### 🎯 学习目标：学会给AI写"工作手册"

AGENTS.md是给AI的开发指令文件，相当于给AI写的“工作说明”，告诉AI在这个项目中应该遵循什么规则。

### 📝 AGENTS.md模板

```Markdown
# AI海龟汤游戏开发指令

## 项目概述
使用React + TypeScript + Tailwind CSS开发的AI海龟汤游戏网站。

## 开发规范
- 使用TypeScript，确保类型安全
- 使用函数式组件 + Hooks
- 使用Tailwind CSS编写样式
- 组件要可复用，代码要有注释

## 代码风格
- 组件名使用PascalCase
- 函数名使用camelCase
- 常量使用UPPER_SNAKE_CASE
- 类型定义以T开头（如TStory）

## 设计要求
- 整体风格：神秘悬疑，深蓝色调（bg-slate-900）
- 强调色：金色（text-amber-400）
- 圆角：rounded-lg
- 阴影：shadow-lg

## 注意事项
- 保持代码简洁，避免过度设计
```

```Plain
Import你的Git仓库(选 GitHub等，授权后选 ai-haigui-game 仓库)。
```

**后端部署（稍复杂，可选）**：

- 可以使用Vercel Serverless Functions / railway，下面使用railway演示。
- 或者保持前端直接调用AI（注意API Key安全）

Railway 后端部署流程

登录并新建项目

- 打开 https://railway.app ，用 GitHub 登录。
- 点 New Project。
- 选 Deploy from GitHub repo。
- 第一次先做 Configure GitHub App，勾选仓库 (你的仓库名)，保存。
- 再选 Deploy from GitHub repo，在列表里点仓库名，点 Deploy Now。

 等第一次构建

- 会从仓库根目录构建（根目录已有 package.json），自动执行 postinstall 在 server 里装依赖，并用 node server/index.js 启动。
- 不需要再设 Root Directory。

配环境变量

- 点进刚建好的那个服务（一张卡片）。
- 左侧或顶部点 Variables。
- 点 + New Variable。
- Name 填：DEEPSEEK_API_KEY
- Value 填：你的 DeepSeek API Key（和本地 server/.env 里一致）。
- 保存

生成公网域名

- 同一服务里点 Settings → Networking。
- 点 Generate Domain。
- 记下生成的地址，例如：https://ai-haigui-game-production.up.railway.app。

### 在 Vercel 里接上后端

- 打开 Vercel 里你的前端项目 → Settings → Environment Variables。
- 新增：Key = VITE_AI_API_URL，Value = 上一步的地址 + /api/chat（例如 https://ai-haigui-game-production.up.railway.app/api/chat）。
